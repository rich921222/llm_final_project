What is Topic Modeling?
●A type of unsupervised machine learning used for text analysis
●Discovers latent topics in a collection of documents without human
intervention
●Helps in organizing, summarizing, and understanding large text corpora
●Unlike classification, topics are not predefined but are derived from patterns in
the data

===== Page 3 =====
Key Topic Modeling Techniques
Latent Dirichlet Allocation (LDA)
●The most widely used topic modeling algorithm
●Assumes each document is a mixture of multiple topics, and each topic is a
mixture of words
●Uses Bayesian probability and Dirichlet distribution to infer topic distributions
●Example use case: Organizing scientific articles into research domains
(overlapping topics, long text)

===== Page 4 =====
Key Topic Modeling Techniques
Non-Negative Matrix Factorization (NMF)
●Uses matrix factorization to find hidden patterns in textual data
●Works well for high-dimensional data by breaking text down into meaningful
components (sparse matrix)
●Does not assume a probabilistic model like LDA (faster)
●Example use case: Extracting themes from customer service chat logs
(non-overlapping topics, short text)

===== Page 5 =====
Key Topic Modeling Techniques
Latent Semantic Analysis (LSA)
●Applies Singular Value Decomposition (SVD) to reduce dimensionality and
capture topic-word relationships
●Good for finding synonym relationships and handling noisy data
●Example use case: Improving search engine results based on contextual
meaning

===== Page 6 =====
LDA Overview
●Probabilistic generative model used for discovering topics in a collection of
documents
●Each document is a mixture of multiple topics, and each topic is a distribution
over words
●Utilizes Dirichlet priors to control topic distributions
●Uses Bayesian inference techniques such as Gibbs sampling or
Variational Inference
●Requires specifying the number of topics (K) beforehand

Key Concepts in LDA
●Documents as Topic Mixtures○Example: A research paper might be 60% about AI and 40% about statistics
●Topics as Word Distributions○Example:
○Topic 1 (Sports): basketball (0.2), football (0.15), tennis (0.1), player (0.05)...
○Topic 2 (Technology): AI (0.2), machine (0.15), learning (0.1), neural (0.05)...
●Dirichlet Distribution○Alpha (α): Controls how documents mix topics
■High α → Documents contain many topics evenly
■Low α → Documents are dominated by fewer topics
○Beta (β): Controls how topics mix words
■High β → Topics share more words
■Low β → Topics are more distinct

===== Page 19 =====
Step-by-Step Process
1.Initialize topics randomly for words in each document (Dirichlet Distribution).
2.Iterative topic assignment using Gibbs Sampling
a.Reassign words to topics based on:
i.How frequently a topic appears in a document
ii.How frequently a word appears in a topic across all documents
3.Repeat until convergence to stabilize topic distributions.
4.Output final results:
a.Topic-word distributions (which words belong to which topics)
b.Document-topic distributions (which topics dominate which documents)

Non-Negative Matrix Factorization (NMF)
●Factorization-based topic modeling method.
●Decomposes a document-term matrix into two lower-dimensional matrices.
●Ensures that all matrix values remain non-negative, making the results easier
to interpret.
●Works well with TF-IDF-weighted text representations.
●Requires predefining the number of topics (K) beforehand.

Choosing the Right Number of Topics
●Setting too few topics: Oversimplifies the data, leading to broad and
uninformative topics
●Setting too many topics: Overfitting, making results harder to interpret
●Methods to determine optimal topic numbers:
○Coherence Score
■Measures how semantically related the top topic words are
■Higher = more meaningful topics
○Perplexity Score
■Measures how well the model predicts unseen data
■Lower = better generalization
■Mainly for LDA

===== Page 53 =====
Coherence Score
●Are top words in a topic meaningfully related?
●Example:○High coherence: apple, banana, grape, orange
○Low coherence: apple, engine, democracy, pillow
●Computed using word co-occurrence or semantic similarity (e.g., C_v, UMass)
●Higher is better
●Good for evaluating topic interpretability

===== Page 54 =====
Perplexity Score
●Measures model's ability to predict unseen documents
●Lower perplexity → better generalization
●Common in language modeling
●But:
○Doesn’t guarantee meaningful or interpretable topics
○A model with low perplexity can still produce incoherent topics
●Not always aligned with human judgment
